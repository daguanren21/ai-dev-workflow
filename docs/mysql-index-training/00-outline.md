# 课程总览

## 一、适合谁学

这套资料适合以下人群：

- 刚接触 MySQL，希望系统理解索引
- 会写 SQL，但不会判断查询是否走索引
- 能看懂一点 Explain，但不会真正优化慢查询
- 面试前想集中梳理 MySQL 索引知识点

## 二、课程目标

完成本课程后，你应该能够：

1. 解释索引和 B+Tree 的关系。
2. 说清楚主键索引、普通索引、聚簇索引、二级索引之间的区别。
3. 判断复合索引是否命中，理解最左前缀原则。
4. 识别覆盖索引、索引下推、回表对性能的影响。
5. 使用 `EXPLAIN` 分析执行计划。
6. 总结并规避常见索引失效场景。

## 五、学习方法

不要只看文档。建议每个知识点都按这个顺序练：

1. 先看 SQL。
2. 自己先猜是否走索引。
3. 再执行 `EXPLAIN`。
4. 对照执行计划分析原因。
5. 修改 SQL 或索引，再验证一次。

## 六、执行计划分析的最小闭环

真正的分析动作只需要下面这个闭环：

1. 明确查询目标：等值查询、范围查询、排序、分页、聚合。
2. 看现有索引：单列索引还是复合索引，列顺序是什么。
3. 预测优化器选择：可能用哪个索引，是否需要回表。
4. 跑 `EXPLAIN`：重点看 `type`、`key`、`rows`、`Extra`。
5. 改索引或改 SQL。
6. 再跑一次验证。

## 七、不要死记硬背的几个点

- 不要把“走索引”误解成“就一定快”。
- 不要把“联合索引”误解成“条件里出现所有字段才能用”。
- 不要把“最左前缀”理解成“只有 where 才能用，order by 就不能用”。
- 不要把 `Using index` 和 `Using index condition` 混为一谈。
- 不要把 `EXPLAIN` 里的 `rows` 当成真实值，它是预估值。
